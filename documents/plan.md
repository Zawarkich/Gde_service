# Сценарии взаимодействия пользователя с программой

### 1. Аутентификация

[Диаграмма аутентификации](./diagramms/Д_аутентификация.png)

*   **Основной поток:**
    2.  Вводит пароль, нажимает "Войти".
    3.  Система отправляет данные на Сервер для проверки.
    4.  Сервер подтверждает данные, создает сессию.

*   **Альтернативный поток (Неверные учетные данные):**
    1.  Сервер отклоняет данные.
    2.  Система отображает ошибку "Неверный пароль".

---

### 2. Управление статусом

*   **Просмотр статусов:**

    [Диаграмма просмотра списка статусов](./diagramms/Д_просмотр_статуса.png)

    1.  Пользователь нажимает "Общий список".
    2.  Сервер возвращает данные о Пользователях и указанные ими статусы

*   **Изменения статусов:**

    [Диаграмма изменения статусов](./diagramms/Д_изменение_статуса.png)

    1.  Пользователь нажимает "Изменить местоположение".
    2.  Система возвращает форму для заполнения.
    3.  Пользователь выбирает один из заготовленных вариантов либо вводит свой, нажимает "Обновить"
    4. Система изменяет статус пользователя в базе данных

---

### 3. Передача прав суперпользователя

*   **Автоматическая передача прав:**

    [Диаграмма передачи прав](./diagramms/Д_передача_прав.png)

    1.  Суперпользователь меняет свой статус на любой, кроме "На месте"
    2.  Система выбирает одного из пользователей без прав суперпользователя и выдаёт ему эти права на временной основе.
    3.  После возврата Суперпользователя Система забирает у временного суперпользователя дополнительные права.
