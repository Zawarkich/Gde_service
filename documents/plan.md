# Сценарии взаимодействия пользователя с программой

### 1. Аутентификация

[Диаграмма аутентификации](./diagramms/Д_аутентификация.png)

*   **Основной поток:**
    2.  Вводит пароль, нажимает "Войти".
    3.  Система отправляет данные на Сервер для проверки.
    4.  Сервер подтверждает данные, создает сессию.
*   **Альтернативный поток (Неверные учетные данные):**
    1.  Сервер отклоняет данные.
    2.  Система отображает ошибку "Неверный пароль".

---

### 2. Управление задачами

*   **Создание задачи:**

    [Диаграмма создания задачи](./diagramms/Д_добавление_задачи.png)

    1.  Пользователь нажимает "Добавить задачу".
    2.  Заполняет форму, нажимает "Сохранить".
    3.  Система отправляет данные на Сервер.
    4.  Сервер валидирует, сохраняет задачу.
    5.  Система обновляет список, отображает новую задачу.
*   **Просмотр задачи:**

    [Диаграмма просмотра задачи](./diagramms/Д_просмотр_задачи.png)

    1.  Пользователь выбирает задачу из списка.
    2.  Система отображает детали задачи.
*   **Удаление задачи:**

    [Диаграмма удаления задачи](./diagramms/Д_удаление_задачи.png)

    1.  Пользователь выбирает задачу, нажимает "Удалить".
    2.  Система отправляет запрос на Сервер, Сервер удаляет задачу.
    3.  Система обновляет список.

---

### 3. Управление проектами

*   **Создание проекта:**

    [Диаграмма создания проекта](./diagramms/Д_добавление_проекта.png)

    1.  Менеджер переходит на страницу проектов.
    2.  Нажимает "Добавить проект", заполняет форму, нажимает "Сохранить".
    3.  Система отправляет данные на Сервер, Сервер валидирует, сохраняет проект.
    4.  Система обновляет список.
*   **Просмотр проекта:**

    [Диаграмма просмотра проекта](./diagramms/Д_просмотр_проекта.png)

    1.  Пользователь выбирает проект из списка.
    2.  Система отображает детали проекта и привязанные задачи.
*   **Удаление проекта:**

    [Диаграмма удаления проекта](./diagramms/Д_удаление_проекта.png)

    1.  Менеджер выбирает проект, нажимает "Удалить".
    2.  Система отправляет запрос на Сервер, Сервер удаляет проект.
    3.  Система обновляет список.

---

### 5. Фильтрация и сортировка

[Диаграмма фильтрации](./diagramms/Д_фильтрация_по_сложности.png)

*   **Основной поток:**
    1.  Пользователь просматривает список задач/проектов.
    2.  Выбирает критерию сложности.
    4.  Система отправляет запрос на Сервер.
    5.  Сервер обрабатывает, возвращает отфильтрованный список.
    6.  Система обновляет отображаемый список.
