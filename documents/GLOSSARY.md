# Glossary

| Термин                            | Определение                                                                                                                                   |
| :-------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------- |
| **Проект (Project)**              | Контейнер верхнего уровня для группировки связанных задач.                                                                                    |
| **Трек (TaskTrack)**              | Список задач внутри проекта, представляющий собой этап или подкатегорию. Задачи в треке могут быть зависимы друг от друга.                    |
| **Расход (Система)**              | Веб-приложение, предназначенное для хранении и предоставления доступа к списку пользователей и их статусам-местонахождениям.                  |
| **Пользователь (User)**           | Участник системы, который имеет права доступа только к своему аккаунту. Может временно наследовать права суперпользователя при передаче прав. |
| **Суперпользователь (Superuser)** | Пользователь, имеющий больше прав доступа, такие как менять статус других пользователей.                                                      |
| **Статус (Status)**               | Местоположение пользователя, которое он указывает в системе.<br>Суперпользователь имеет права менять статусы других пользователей.            |
| **Spring Boot**                   | Основной фреймворк на языке Java, используемый для создания серверной части (backend) приложения.                                             |
| **Thymeleaf**                     | Серверный шаблонизатор, который используется для генерации HTML-страниц и динамического отображения данных.                                   |
| **Spring Data JPA**               | Технология, упрощающая взаимодействие с базой данных путём автоматического создания запросов на основе Java-объектов (сущностей).             |
| **H2 Database**                   | Встраиваемая реляционная база данных, используемая в проекте для хранения всей информации (задач, проектов, треков).                          |
| **Maven**                         | Инструмент для автоматизации сборки проекта и управления его зависимостями (библиотеками).                                                    |
| **Эндпоинт (Endpoint)**           | Конкретный URL-адрес на сервере, к которому обращается клиент для выполнения действия.                                                        |
| **Контроллер (Controller)**       | Компонент Spring, отвечающий за приём и обработку HTTP-запросов от пользователя и возврат результата.                                         |
| **Модель (Model/Entity)**         | Java-класс, описывающий структуру данных, хранимых в базе данных.                                                                             |
| **Репозиторий (Repository)**      | Интерфейс в Spring Data JPA, предоставляющий стандартные методы для работы с базой данных.                                                    |
| **Безопасность (Security)**       | Набор мер, реализованных с помощью Spring Security, для защиты приложения и разграничения доступа.                                            |
